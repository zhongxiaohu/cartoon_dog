开发环境

根据官方指引，它推荐我们使用 idea 去做 Scala 开发，因此我在 idea 中安装相关插件（截图在后面），同时本机环境我们也得去安装一下，方便后续系统调用。

注意版本：推荐 2.11.12

地址：https://www.scala-lang.org/download/2.11.12.html

疯狂下一步，安装完后 重启电脑 或者按 win+r 输入cmd

注意：一定要win+r

下面是 idea 安装 scla 插件截图，idea 与 pycharm 师出同源，更适合我们做开发。

创建 maven 项目，勾选 scala 案例

可能会出现插件安装失败，例如：

​x
Cannot resolve plugin org.scala-tools:maven-scala-plugin:<unknown>

在依赖中添加

<dependency>
    <groupId>org.scala-tools</groupId>
    <artifactId>maven-scala-plugin</artifactId>
    <version>2.11</version>
</dependency>
<dependency>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-eclipse-plugin</artifactId>
    <version>2.5.1</version>
</dependency>

centos虚拟机

三台虚拟机，推荐使用 VirtualBox，不推荐 VMware

centos7下载：阿里云镜像站点 ，选择 CentOS-7-x86_64-DVD-2009.iso ，下载速度几乎跑满。

改一下名称和文件夹路径，剩下的配置全部默认即可。

右键虚拟机，选择 分离式启动，选择 iso镜像 文件路径

这一步不知道是点注册还是创建，我点注册后选择镜像，发现需要重启虚拟机才可以安装centos7，应该是点击注册没错。慢慢等待它安装完毕。出现ui界面后，选择中文，然后点击安装位置，然后进去直接点左上角完成，出来后点开始安装

安装过程中会让你选择root密码，推荐使用123456，需要按两次完成，其他不用动。

系统安装完以后，一定要检查有没有java环境，有的话要卸载掉！

输入ip addr 查看虚拟机ip，发现此时系统并没有联网，我们得去该虚拟机的设置中配置网络。

理论上应该是可以的，可就是没有ip地址，猜测网卡并没有启动。

于是我们可以看一下网卡配置，根据返回显示，我虚拟机网卡为enp0s3

查看该网卡配置

x
 vi /etc/sysconfig/network-scripts/ifcfg-enp0s3

果然网卡并没有启动。。。

把no改成yes，重启服务

x
service network restart

终于ip出来了，终于可以摆脱这垃圾shell了，我们使用自己喜欢的shell工具去连接虚拟机即可，ssh工具大同小异。用户名root，密码123456

此时的shell工具连不上虚拟机，我们还需要增加一个网卡来仅主机访问。

查看该网卡ip，然后在工具中连接。

为了跟视频保持安装目录一致方便后续操作，我们也去创建跟他一样的文件夹。

x
[root@localhost ~]# mkdir -p /data/program/software
[root@localhost ~]# cd /data/program/software/

将jdk1.8拉过去，解压，设置环境变量即可。

xxxxxxxxxx
tar -zxvf 文件名
vi /etc/profile
​
#下面的内容加到底部
#java environment
export JAVA_HOME=/data/program/software/jdk1.8.0_271
export CLASSPATH=.:${JAVA_HOME}/jre/lib/rt.jar:${JAVA_HOME}/lib/dt.jar:${JAVA_HOME}/lib/tools.jar
export PATH=$PATH:${JAVA_HOME}/bin
​
#刷新变量
source /etc/profile
​
#测试jdk环境
java -version

host配置

做这一步之前，我们把另外两台虚拟机给复制出来，改一下名字和路径，其他默认选项即可

xxxxxxxxxx
vi /etc/hosts
​
#在底部加入下面的信息，你的ip可能跟我不一样，注意。三台虚拟机都要。

关闭防火墙

centos7之后的命令主要以 firewall 为主

x
#查看防火墙状态
firewall-cmd --state
#停止firewall
systemctl stop firewalld.service
#禁止firewall开机启动
systemctl disable firewalld.service
